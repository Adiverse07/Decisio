# Constraints

Hard rules that apply to every change in Decisio, whether written by a human or generated by AI.

## Database Constraints

1. **No raw SQL.** All queries go through SQLAlchemy ORM. No `db.session.execute("SELECT ...")`.
2. **No hard deletes.** Decisions use `is_deleted` soft-delete flag. No `db.session.delete()` on decisions.
3. **Migrations track all schema changes.** Column additions, constraint changes, and index modifications go through Flask-Migrate. No manual `ALTER TABLE`.
4. **CHECK constraints are enforced at the database level.** `no_self_supersede` and `supersede_status_sync` exist in the Decision model and must not be removed.

## Authentication & Authorization

5. **JWT tokens are required for all non-auth routes.** Every route outside `/api/auth/*` uses `@require_login`. No anonymous access.
6. **Admin routes use `@require_admin`.** User management (create, list, delete) and decision deletion are admin-only. This is enforced by middleware, not by frontend checks.
7. **Passwords are never stored in plain text.** Flask-Bcrypt handles hashing. Passwords are never returned in API responses.
8. **JWT secret comes from environment variables.** Never hard-coded.

## Business Logic

9. **Status transitions are forward-only.** The `STATUS_TRANSITIONS` dict in the Decision model defines allowed transitions. `proposed → active → decided → completed`. No backward moves.
10. **Superseded decisions cannot be edited.** Once a decision's status is `superseded`, it is immutable except for the `is_deleted` flag.
11. **Every state change is audited.** Status transitions, supersede actions, and deletions create an `AuditLog` entry with the user, action, and timestamp.
12. **Validation happens in schemas, not in routes.** Marshmallow schemas validate all input. Routes call `schema.load(data)` and trust the result.

## Frontend Constraints

13. **No direct API calls from components.** All API access goes through `lib/api.ts`. Components use `DataContext` to access data and actions.
14. **Types are defined once in `lib/types.ts`.** No inline type definitions that duplicate what's already in the types file.
15. **UI components come from `components/ui/`.** No installing additional UI libraries. ShadCN + Radix covers the component needs.

## Testing Constraints

16. **Tests use a separate database (`decisio_test`).** Test config points to a different database. Tests never touch `decisio_dev`.
17. **Each test is independent.** Tests create their own data via fixtures. No test depends on another test's side effects.
18. **All 36 tests must pass before any commit.** Run `pytest -v` from the `backend/` directory. Zero failures, zero errors.

## Dependency Constraints

19. **No new backend dependencies without justification.** The current stack (Flask, SQLAlchemy, Marshmallow, PyJWT, Bcrypt) covers the requirements. Adding a package requires a reason.
20. **No new frontend dependencies without justification.** ShadCN, Radix, Tailwind, Recharts, and React Hook Form cover the UI needs.
